Objective:

This lab involves writing and testing various methods.  The methods will be documented using Javadoc comments.  All the methods will be written as static methods.

Background:

This lab is based on material in Section 4.4 (The Anatomy of a Method) from the textbook.
 

Instructions:

  1. Launch the Eclipse development environment and create a new project named Lab 8.  Place a copy of Fun.java into the src folder of the project.  (Either download the file to the Desktop and then copy it into Eclipse or copy it from the file server.)  Then also create a new file FunctionTester.java (with a main method) that you will use to verify that the various functions in this lab work correctly.

    Examine the Fun class.  How many methods does it contain?  For each method, give its signature and show a "sample call" to that method.


  2. Add statements to the main method in FunctionTester to test the sumOfFactors() method in the Fun class.  Your statements should include both a description of the input and the output so that it is obvious from reading the statement that the answer is correct. Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the statements used to test the method and turn these in as part of your lab.


  3. Add statements to the main method in FunctionTester to test the passFail() method in the Fun class.  (You may replace the statements from Step 2.)  Your statements should include both a description of the input and the output so that it is obvious from reading the statement that the answer is correct. Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the statements used to test the method and turn these in as part of your lab.


  4. Write a static method that calculates the sum 1 + 2 + 3 + . . . + n.  Add Javadoc comments to the method.  Write statements in the main method to test this method.  (In this step, and each of the following steps, you may replace the statements from the previous step.)  Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and print the statements used to test the method and turn these in as part of your lab.


  5. Write a static method that finds the largest of five numbers.  Add Javadoc comments to the method.  Write statements in the main method to test this method.  Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and print the statements used to test the method and turn these in as part of your lab.


  6. Write a static method that converts an integer to a month name, e.g., 1 --> January.  Add Javadoc comments to the method.  Write statements in the main method to test this method.  Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and print the statements used to test the method and turn these in as part of your lab.


  7. Write a static method that finds the number of e's in a string.  Add Javadoc comments to the method.  Write statements in the main method to test this method.  Use at least three cases to test your method.

    Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and print the statements used to test the method and turn these in as part of your lab.


  8. Write a static method that uses a loop to sum the first k odd integers.  (Thus, if k=4, the sum is: 1+3+5+7.)  Add Javadoc comments to the method.  Write statements in the main method to test this method.

    Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and print the statements used to test the method and turn these in as part of your lab.


Extra Credit:  (two points) 

Rewrite the static method from Step 8 so that it does not use a loop!  (This should result in a much shorter method!).  Also, present a clear and convincing argument that your new method always returns the same answer as the one in Step 8.

Run the program that tests the method.  Print the output from the test.  Print the method (including the comments) and turn these in as part of your lab.  (You can skip the testing statements; I should have them from Step 8!)
