Objective:

This lab introduces the if statement and the for statement in the context of rolling dice.

 

 

Background:

Section 5.2 of Lewis & Loftus introduces the if statement.  Section 5.8 of Lewis & Loftus introduces the for statement.  This lab uses the Dice class from Lab 2.

 

 

Instructions:

1.     Launch the Eclipse development environment. Create a new project named Lab6. 

Add the file Dice.java to your project.  You can use the link earlier in this paragraph, you can find this file in CSCourses \ CS131 \ Lab6. or you can copy/paste the file from your Lab 2 assignment.  (Dice.java defines the same Dice class as in Lab 2.)

Create a main class named DiceMain.  Within the main method create two 6-sided Dice objects, roll each of the dice, and calculate the sum of the roll values.  Use the console (System.out.println) to display the values of the two dice rolls and their sum.

Test the program to be sure your code is correct.

 

2.     Modify the program DiceMain so that it requests and obtains a value from the user (the user’s “guess”) then rolls the two dice and displays a message based on the sum of the roll values:
          “You win!”  If the sum matches the user’s guess.
and
          “You lose!  The correct answer was:  X”  If the sum does not match the user’s guess.

Use an if statement to compare the sum with the user’s guess.  The statement should have the form:
     if (sum == guess)
  {
     <display the message: “You win!”>
  }
  else
  {
    <display the message:
                   “You lose! The correct answer was: [sum]”> 

   // Where [sum] is replaced by the actual sum
  }

Note: Use the Scanner class to input the user's guess.

Include a copy of the statements you used to accomplish this in your lab write-up.  (You may merely provide a copy of the body of the main method.)

 

3.     Modify the DiceMain program to play a different game.  The program will not obtain a value from the user.  Instead the program will roll the dice and declare:
-     “You win!” if the sum is greater than 7
-     “You lose!”  if the sum is less than 7
-     “There is a tie.” if the sum is 7
In addition to the message the program should display the value of the sum.

Run the modified program enough times to test each of the three cases.

Include a copy of the statements you used to accomplish this in your lab write-up.

 

4.     Modify the DiceMain program so that it appears as it did in Step 1 where the two dice were rolled and the values of the rolls and the sum were displayed.  Modify this program to roll the two dice 20 times, displaying the values of the rolls and sum each time.  To do this enclose the statements to roll the dice and display the values of the rolls and the sum in a for loop.  The for statement has the form:
  for (int i = 0; i < 20; i++)
  {
     <statements to be repeated 20 times>
  }

Include a copy of the statements you used to accomplish this in your lab write-up.

 

5.     Modify the DiceMain program to roll two dice 20 times and count the number of times a 7 is rolled.  To do this introduce a variable named count7 to keep track of the number of 7’s.
-     What type of variable should count7 be?
-     What initial value should be assigned to count7?

Declare and initialize the variable count7 before the for loop.

To count the number of 7’s, after the sum of the rolls is calculated inside the for loop, check if the sum is 7.  If it is 7 add 1 to the variable count7.  You can use a simple if statement of the form:
     if (<condition>)
  {
     <do what is necessary>
  }
to do this.  Display the number of 7's after the for loop terminates.  (Keep the code to display the values of each roll and sum so you can verify that your code is correct.)

Include a copy of the statements you used to accomplish this in your lab write-up.

 

6.     Modify the program DiceMain to roll the dice 10,000 times and count the number of 7’s.  Comment out the code that displays the values of each roll and sum within the loop. 

Run the program five times and record your results. 

In your lab write-up determine the mathematical probability of rolling a seven.  (Hint: there are 36 possible outcomes of the rolls: 1-1, 1-2, 1-3, . . . , 2-1, 2-2, . . .  Each of the 36 outcomes is equally likely.  Of the 36 possible outcomes how many of them yield a seven?)  Compare this number with the number of sevens rolled in each of the five runs.  The comparison provides a measure of the quality of the random number generator that is used in Java. 

 

7.     Finally modify the program DiceMain to ask the user for the number of times to roll the dice then roll the dice the number of times requested and count the number of sevens.  Be sure to display the values of each roll.

Add a copy of the output from executing this program to your report.  Document your code.   Print a copy of this program and hand it in with your lab.
